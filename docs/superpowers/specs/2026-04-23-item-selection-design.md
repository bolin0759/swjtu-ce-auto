# SWJTU Course Python 复选执行设计（V1.1）

- 日期：2026-04-23
- 范围：仅改动 `swjtu-course-python` 的 UI 选择层，不改浏览器执行核心逻辑。

## 1. 目标与验收

### 目标
在“当前课程筛选结果”区域支持逐条勾选执行，满足手动测试少量条目（如仅 5 条）。

### 验收标准
1. 刷新列表后默认全不勾选。
2. 支持快捷按钮：全选 / 全不选 / 反选。
3. 点击开始仅执行勾选条目；未勾选条目全部跳过。
4. 若未勾选任何条目，给出提示并不启动任务。
5. 进度条分母按“勾选条目数”计算；日志显示“已选择 X/Y 条”。

## 2. 约束与非目标

### 约束
- 保持现有 Playwright 单线程工作队列模型不变。
- 不引入新依赖。
- 不改动课程抓取与条目筛选规则（仍由现有筛选条件决定列表内容）。

### 非目标
- 本次不实现“前N条自动选择”。
- 本次不实现跨课程复选框。
- 本次不做列表排序/搜索增强。

## 3. 方案对比与选择

已评估方案：
1. CTkScrollableFrame + CTkCheckBox（选用）
2. ttk.Treeview 勾选列
3. 文本框 + 手工输入索引

选型理由：方案1改动最小、风格一致、满足当前需求。

## 4. 详细设计

### 4.1 UI结构改动（`ui/main_window.py`）
- 将 `self.items_text`（CTkTextbox）替换为：
  - 顶部操作栏按钮：全选、全不选、反选
  - 可滚动容器 `CTkScrollableFrame`
- 每条数据渲染为一行复选框：
  - 文案格式：`001. [doc] [未学习] 标题`
- 新增状态容器：
  - `self.item_check_vars: list[ctk.BooleanVar]`
  - `self.item_rows: list[CourseItem]`

### 4.2 默认选择行为
- `_render_items()` 每次刷新后：
  - 清空旧控件
  - 重新创建复选框，默认 `False`

### 4.3 快捷选择动作
- `on_select_all()`：全部置 `True`
- `on_select_none()`：全部置 `False`
- `on_select_invert()`：逐项取反

### 4.4 开始执行逻辑
- `on_start()` 中新增：
  - 从 `item_check_vars` 收集勾选索引
  - 根据索引映射出 `selected_items`
  - `len(selected_items)==0` 时日志警告并返回
  - 启动 `runner.start(selected_items, ...)`
- 日志：`已选择 X/Y 条，开始执行`

### 4.5 进度与日志
- 进度条分母自然使用传入 runner 的 `selected_items` 数量。
- 其余执行/暂停/继续/停止逻辑不变。

## 5. 错误处理

- 未连接浏览器时保持现有报错行为。
- 条目列表为空时保持现有提示逻辑。
- 勾选列表为空时新增明确提示，避免“点开始没反应”。

## 6. 测试计划

手工验证：
1. 连接课程页后刷新，确认默认全不勾选。
2. 点“全选”后开始，确认执行数量=列表数量。
3. 点“全不选”后开始，确认阻止执行并给提示。
4. 点“反选”两次，确认状态可逆。
5. 手工只勾 5 条开始，确认进度分母=5。
6. 执行期间暂停/继续/停止，确认行为与当前版本一致。

- 仅变更：`swjtu-course-python/ui/main_window.py`
- 不变更：`core/browser_controller.py`、`core/task_runner.py` 及数据模型
