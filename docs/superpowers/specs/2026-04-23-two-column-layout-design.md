# SWJTU Course Python 两栏布局设计（V1.2）

- 日期：2026-04-23
- 范围：仅调整 `swjtu-course-python/ui/main_window.py` 的布局组织，不改变业务行为。

## 1. 目标与验收

### 目标
将“筛选结果”和“日志”改成并列两栏，提升同时观察与操作效率。

### 验收标准
1. 顶部区域（连接/筛选/控制/进度）位置保持不变。
2. 下方内容区改为并列两栏：左筛选结果、右日志。
3. 左右宽度比例为 60/40。
4. 窗口缩小时两栏保持并列，不自动换行。
5. 两栏内部保持各自滚动能力。

## 2. 约束与非目标

### 约束
- 只改布局，不改执行逻辑。
- 不改 BrowserController / TaskRunner。
- 不引入新依赖。

### 非目标
- 不调整按钮文案与功能。
- 不改主题、字号、颜色。
- 不做自适应断点切换（如窄屏改单列）。

## 3. 方案对比与选择

### 方案A（选用）：新增内容双栏容器
- 做法：将 row=4、row=5 重构为一个内容容器 frame（放在 row=4），内部两列 60/40，左放筛选结果子框，右放日志子框。
- 优点：结构清晰，后续维护成本低，和现有顶部区域解耦。

### 方案B：直接让现有 items/log frame 同行并列
- 优点：改动行数更少。
- 缺点：与现有 row 语义耦合重，后续扩展容易混乱。

### 方案C：运行时动态重排
- 优点：兼容旧结构。
- 缺点：复杂且无必要。

结论：采用方案A。

## 4. 详细设计

### 4.1 主网格
- 当前主窗体 `grid_rowconfigure(4, weight=1)`、`grid_rowconfigure(5, weight=2)` 调整为：
  - row=4 作为唯一内容区（weight=1）
  - row=5 不再承载日志区

### 4.2 内容双栏容器
- 新增 `self._build_content_frame()`：
  - 容器放在 `row=4, column=0, sticky='nsew'`
  - `grid_columnconfigure(0, weight=3)`（左 60）
  - `grid_columnconfigure(1, weight=2)`（右 40）
- 左侧调用 `_build_items_frame(parent, row=0, col=0)`
- 右侧调用 `_build_log_frame(parent, row=0, col=1)`

### 4.3 子构建函数签名调整
- `_build_items_frame()` 与 `_build_log_frame()` 改为接收父容器参数，避免写死到主窗体。
- 两个子框内部继续各自 `sticky='nsew'`，保证内部滚动控件可扩展。

### 4.4 并列保持策略
- 不引入响应式断点。
- 通过固定两列与权重控制，在窗口缩小时仍保持左右并列；视觉拥挤时依赖内部滚动。

## 5. 测试计划

1. 启动程序，确认顶部四块区域位置不变。
2. 确认下方为左右并列：左筛选结果、右日志。
3. 调整窗口宽度到较窄，确认仍并列不换行。
4. 刷新条目并执行，确认左侧列表滚动正常，右侧日志持续输出并滚动正常。
