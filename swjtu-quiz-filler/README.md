# SWJTU 答题助手

自动读取西南交通大学网络学堂客观题，并根据答案文件完成选择操作。

---

## 功能

- 连接已打开的 Chrome 浏览器（通过 CDP 协议）
- 扫描当前答题页面，识别所有单选题和多选题
- 按行读取答案文件，自动点击对应选项
- 完成后可手动验证选中状态，确认无误后自行提交

---

## 环境要求

- Python 3.10+
- Google Chrome

---

## 安装

```bash
pip install -r requirements.txt
playwright install chromium
```

---

## 使用步骤

### 1. 准备答案文件

每行对应一道题，第 N 行 = 第 N 题：

```
存储程序与自动控制
通用计算机和专用计算机
由于计算机的可编程性，计算机能够在各行各业得到广泛的应用
...
内存储器；I/O设备；中央处理器
```

- **单选题**：直接写选项文字
- **多选题**：用 `；`（中文分号）分隔多个选项
- **填空题 / 主观题**：留空，程序自动跳过

### 2. 启动程序

```bash
python app.py
```

### 3. 操作流程

```
一键启动并连接  →  加载答案  →  扫描页面  →  开始答题  →  验证结果  →  手动提交
```

| 按钮 | 说明 |
|------|------|
| 一键启动并连接 | 自动检测 Chrome 路径，以调试模式启动并连接 |
| 连接浏览器 | 连接已手动开启 CDP 端口的 Chrome |
| 加载答案 | 选择答案文件并解析 |
| 扫描页面 | 读取当前页面的所有单选/多选题 |
| 开始答题 | 按答案逐题点击选项 |
| 验证结果 | 重新读取页面选中状态，标注 ✓ / ✗ |

## 项目结构

```
swjtu-quiz-filler/
├── app.py                    # 入口
├── requirements.txt
├── core/
│   ├── models.py             # 数据模型
│   ├── logger.py             # 日志工具
│   ├── answer_parser.py      # 解析答案文件
│   ├── browser_controller.py # CDP 连接、页面扫描、点击操作
│   └── task_runner.py        # 多线程任务控制
└── ui/
    └── main_window.py        # GUI 主窗口
```

---

## 注意事项

- 程序只负责**选择**，不会自动提交，请人工确认后再点提交
- 答案文件的行数须与题目顺序一致
- 适用于基于 Element UI 构建的答题页面（`div.list-item` 结构）
